Practice your knowledge on test or non-production server

  • Create a test database that has the same structure as your production database, but with only a few representative rows in the tables.

  • Run each test query on the test data and see whether the results match your exceptions.

  • Build several sets of test data and be sure to include odd cases, such as;

    • empty tables

    • extreme values at the very limit of allowable ranges

  • Always try to think of unlikely scenarios and check for proper behavior when they occur

  • The best part is that in the course of checking for unlikely cases, you will be amazed to see several insights into problems that are more likely to happen in the real production database.
